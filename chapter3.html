3. Core Methods

This section contains the methods that the user will use to implement the basic functionality of the package.
The methods cover everything from training an LSTM model and perform imputations for missing data. It also contains supporting methods that calculates an analytical and visual description of the data.


3.1 Training the Model

This method is used to train the LSTM imputation model on data. The default data set for the training includes data from the CGMAnalysis package, Gluvarpro package, and the CGMAnalyzer package. The user may specify their own data if they wish to.

Method Signature: def train(self, data)

Function Parameters:

Name Description Datatype
data the dataset (CSV file) entered by the user to train the model. It should have the following format:
Display Time -> object
GlucoseValue -> float64
subjectId -> object
 pandas Dataframe


Return:The output is a model trained on the supplied data that can be used to perform imputations imputations

Example: Training the LSTM model




3.2 Impute Data

This method performs the imputations on gaps present in the individual's glucose values using the trained LSTM model. It saves the imputed values in a directory and displays the path for the file. The function also returns a visual representation of the imputed data.

Method Signature: def impute(self, test_data, flag)

Function Parameters:

Name Description Datatype
test_data the dataset (CSV file) entered by the user with a gap that needs to be imputed. It should have the following format:
Display Time -> object
GlucoseValue -> float64
subjectId -> object
 pandas Dataframe
flag A flag variable to decide whether to return the imputed values as a pandas data frame (flag=1) or to save the imputed values as a CSV file and plot the graph for it (flag=0). integer


Return: The imputed values as a pandas data frame (flag=1) or the imputed values saved as a csv file and a line graph for it

Example: Getting the imputed values



Example: Comparison of real and predicted values




3.3 Describe Data

This method provides a statistical description of the default data used for training the model in the form of a consolidated table. This data has been trimmed to have only complete days with no missing values. This description table is saved as a CSV file for future reference.

Method Signature: def dataDescribe(self, data, meta)

Function Parameters:

Name Description Datatype
data The default dataset (CSV file) includes data from the CGMAnalysis package, Gluvarpro package, CGMAnalyzer package and the Ohio University with the following format:
Display Time -> object
GlucoseValue -> float64
subjectId -> object
 pandas Dataframe
meta The dataset (CSV file) with the metadata about the status of the individuals in the following format:
ID -> object
status -> object
 pandas Dataframe


Return: A tabular and graphical representation of the statistical analysis of the consolidated data. This table is also saved as a csv file.

Example: A statistical description of the data

3.5 Plot Individual

This method plots the graph of the Glucose Values of a single Subject ID. The subject ID can be found as a part of the description of the data. The default data includes data from the CGMAnalysis package, Gluvarpro package, and the CGMAnalyzer package dataset. The user may specify their own data if they wish to. The subject ID is a part of the data supplied by the user.
